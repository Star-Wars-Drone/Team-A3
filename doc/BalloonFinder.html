These are the most relevant external functions of BalloonFinder() located in balloon_finder.py()

Note:

Before Balloon_finder can work properly, the camera needs to be calibrated.

First using imutils/bin/range-finders, you must determine the hsv threshholds by changing the slider values until only the target object is visible.

Set these values in self.lower_red, self.upper_red

Next to calibrate the camera, see the OpenCV-camera-Calibration file linked in software.

You must print out the checkerboard pattern, and then modify the default.xml input file with the relevant information (such as the ‘size’ of the pixel, this is where the units of the output vectors are defined).

Next just run the calibration file with ./CalibrateCamera default.xml out.xml

Input the parameters found in out.xml into the self.cam_matrix and self.distcoeffs matrices.

Examples of calibration files are found inside of the data folder.

find_balloons():

         Finds list of all balloon-like contours in image

            recommended to use an image that has masked out all non-red

            :return: list of contours suspected of being balloon.

pick_best_balloon(balloon_list):

Receives list of contours it suspects to be balloons and determines which one is

                most likely to be the true balloon.

is_balloon(contour):

        Returns true or false on whether or not the contour is suspected of being a balloon.

        Low strictness, occasional false positives.

is_definitely_balloon(contour):

        True or false with high confidence, other red objects are not seen as balloons.

save_image():

        Saves most recent frame with the name `IM_CAP_<number>`

        The name is incremented to avoid overwriting.

find_vector(contour):

Finds the displacement vector to a balloon of known size in inches.

find_waypoint(current_location, contour):

        Given a current location, and the balloon contour, it calculates the the co-ordinates of where the balloon would be relative to that location.
